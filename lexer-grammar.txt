Building My Own Lexer

# What should be implemented : 
	- Prompt
	- History
	- Signals : ctrl+D, ctrl+C, ctrl+\
	- Builtins : pwd, ...
	- Env-vars : env, ...
	- Pipe
	- Redirection
	- quoted_string
	- Exectuable

# RegEx that matches these tokens types : 
	OPERATOR: \|{1,2}|&{1,2}|;|\(|\)   
	command  : [a-zA-Z][a-zA-Z0-9_-]*
	ARGUMENT : [^\s|&;]+
	Prompt : ^[$#]\s*

# Implementation of a Regex-based lexer :
	1. Read the command-line from input.
	2. Pass it to the lexer for tokenization.
	3. Pass the tokens to the parser.

# Functions we need inside lexer.c file : 
	1. t_token **lexer(int ac, char **av) ==> this func is going to return a 2d array that holds
	 info the following way (let's take ls command as an example)

			t_token tokens[] = 
			{
				{CMD, "ls"},
				{OP, "-"},
				{ARG, "l"),
				{UKNOWN, NULL} // End of Array
			};

--->   This will be the input of the parser.
